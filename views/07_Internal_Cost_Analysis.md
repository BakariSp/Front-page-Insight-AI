# 內部成本與利潤分析報告 (Internal Cost Analysis)

**日期**：2026-01-05
**版本**：v2.0 (已依據銷售方案 A/B/C 修正)
**基礎模型**：GLM-4.7 (參考截圖定價)
**硬件標準**：RTX 4090 / Booster K1

---

## 1. 學校場景用量測算 (Usage Estimation)

基於用戶提供的參數及多學科場景修正：
*   **學校規模**：600 人
*   **交互模式**：
    *   **視覺模型 (Vision)**：OCR 識別手寫作業及試卷。
    *   **Realtime 模型**：AI 助教語音互動。
*   **年度總多模態 Token 用量**：**6 億 Tokens / 年** (此為高價值算力)。

---

## 2. 內部 AI 成本測算 (Total Operation Cost)

**A. API 流量與算力成本 (Variable Cost)**：
*   **視覺與實時模型消耗**：考慮到 6 億 Tokens 的規模，且包含高價 Vision/Audio 模型。
*   **年度預算**：**20,000 RMB / 校** (約 HK$ 21,600)。

**B. 平台運維與開發攤分 (Fixed Cost)**：
*   **雲端運維 (Maintenance)**：HK$ 5,500 / 年。
*   **持續開發 (Dev Cost)**：HK$ 5,400 / 年。

**C. 年度總運營成本合計**：
$$
\approx \text{HK\$ 32,500 / 校 / 年}
$$

> **💡 分析結論**：雲端版方案 (A/B) 需承擔此持續性成本，而本地版方案 (C) 可大幅節省此項開支。

---

## 3. 硬件成本與利潤結構 (Margin Analysis)

**基礎成本假設 (COGS)**：
1.  **Booster 機器人硬件成本**：**HK$ 43,000 / 台** (內部調貨價)。
2.  **高性能運算服務器 (4x4090)**：**HK$ 115,000 / 套** (含服務器工作站 + 4張卡 + 組裝)。
3.  **物流與部署 (Logistics)**：HK$ 2,000 - 8,000 (視方案複雜度)。
4.  **大模型優化與微調 (LLM Ops)**：HK$ 10,000 / 項目 (僅限方案 C)。

### 📊 方案 A：AI 啟動體驗包 (HK$ 48,000)
*   **收入**：HK$ 48,000
*   **直接成本 (COGS)**：
    *   機器人 x 1：HK$ 43,000
    *   物流與部署：HK$ 2,000
    *   物料與培訓：HK$ 2,000
    *   平台成本 (3個月)：HK$ 8,125
    *   *總成本*：**HK$ 55,125**
*   **毛利 (Gross Profit)**：**- HK$ 7,125 (負毛利)**
*   **評價**：**戰略性虧損**。作為獲客入口，嚴格控制名額。

### 📊 方案 B：AI 教學無憂計劃 (HK$ 188,000)
*   **收入**：HK$ 188,000
*   **直接成本 (COGS)**：
    *   機器人 x 2：HK$ 86,000
    *   物流與部署 (2台)：HK$ 4,000
    *   維護儲備 (1.5年)：HK$ 4,500
    *   物料與培訓：HK$ 2,000
    *   平台成本 (1.5年)：HK$ 48,750 (HK$ 32,500 x 1.5)
    *   *總成本*：**HK$ 145,250**
*   **毛利 (Gross Profit)**：**HK$ 42,750**
*   **毛利率**：**22.7%**
*   **評價**：**微利推廣**。利潤空間被高昂的 Token 成本壓縮，核心價值在於 1.5 年後的續費轉化。

### 📊 方案 C：AI 本地化私有雲方案 (HK$ 198,000)
*   **收入**：HK$ 198,000
*   **直接成本 (COGS)**：
    *   **硬件成本 (HW)**：**HK$ 115,000**
        *   *4x RTX 4090 + Threadripper Workstation*
    *   **部署與實施 (Deployment)**：**HK$ 10,000**
        *   *專人上門、私有雲環境搭建、網絡調試*
    *   **大模型優化 (LLM Fine-tuning)**：**HK$ 10,000**
        *   *針對性 Prompt 優化、校本知識庫嵌入*
    *   **軟件授權與加密狗**：HK$ 2,000
    *   **首年維護 (SLA)**：HK$ 5,000 (預留備件與人力)
    *   *總成本*：**HK$ 142,000**
*   **毛利 (Gross Profit)**：**HK$ 56,000**
*   **毛利率**：**28.3%**
*   **評價**：**穩健收益**。
    *   雖然硬件與人力投入大，但**無需承擔後續的高昂 Token 流量費**。
    *   次年維護費 (HK$ 39,800) 幾乎為純利。
    *   相比方案 B，風險更可控 (不受 Token 用量波動影響)。

---

## 4. 關鍵建議

1.  **方案 C 利潤更健康**：雖然方案 C 毛利率 (28%) 與方案 B (23%) 相近，但方案 C **沒有流量超支風險**。方案 B 如果學校用量過大，利潤會進一步被 Token 成本吞噬。
2.  **嚴控方案 A**：每賣出一套方案 A 虧損 7 千元，必須設定為「僅限首 5 間種子學校」或「每區限額」。
3.  **推廣重點**：
    *   對預算敏感但想長期用的學校 -> **推方案 C** (一次性投入，無後顧之憂)。
    *   對即時教學效果要求高，不想管硬件的學校 -> **推方案 B**。
